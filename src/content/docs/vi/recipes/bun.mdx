[Bun](https://bun.sh/) là một môi trường chạy JavaScript & bộ công cụ tất cả trong một. Xem [tài liệu của Bun](https://bun.sh/docs) để biết thêm thông tin.

:::caution
Sử dụng Bun với Astro có thể gặp một số vấn đề chưa hoàn thiện. Một số tích hợp có thể không hoạt động như mong đợi. Tham khảo [tài liệu chính thức của Bun về làm việc với Astro](https://bun.sh/guides/ecosystem/astro) để biết chi tiết.

Nếu bạn gặp bất kỳ vấn đề nào khi sử dụng Bun, vui lòng [mở Issue trên GitHub với Bun trực tiếp](https://github.com/oven-sh/bun/issues/new/choose).
:::

## Yêu cầu trước

- Bun được cài đặt cục bộ trên máy bạn. Xem [hướng dẫn cài đặt](https://bun.sh/docs/installation) trong tài liệu chính thức của Bun.

## Tạo một dự án Astro mới với Bun

Tạo một dự án Astro mới với Bun bằng lệnh `create-astro` sau:

```bash
bunx create-astro@latest my-astro-project-using-bun
```

:::tip
Bạn cũng có thể [tạo một dự án Astro mới từ bất kỳ kho GitHub Astro hiện có nào](/vi/install-and-setup/#install-from-the-cli-wizard) bằng cờ `--template`:
```bash
bunx create-astro@latest my-astro-project-using-bun --template eliancodes/brutal
```
:::

## Cài đặt các phụ thuộc

Nếu bạn bắt đầu một dự án mới bằng `bunx create-astro`, CLI sẽ tự động sử dụng Bun để cài đặt các phụ thuộc và bạn có thể bỏ qua bước này.

Nếu không, bạn sẽ cần cài đặt các phụ thuộc của mình với Bun:

```bash
bun install
```

## Thêm Types

Bun phát hành gói [`@types/bun`](https://www.npmjs.com/package/@types/bun), chứa các kiểu runtime cho Bun.

Cài đặt `@types/bun` với lệnh sau:

```sh
bun add -d @types/bun
```

## Sử dụng các tích hợp Astro

Bạn cũng có thể sử dụng bất kỳ tích hợp chính thức nào của Astro với lệnh `astro add`:

```bash
bunx astro add react
```

## Chạy Astro với Bun

:::note
Sử dụng [cờ CLI `--bun`](https://bun.sh/docs/cli/bunx#shebangs) trước mỗi lệnh `astro` để sử dụng runtime của Bun thay cho Node.
:::

### Chạy máy chủ phát triển

Để chạy máy chủ phát triển sử dụng Bun làm runtime, dùng lệnh sau:

```bash
bunx --bun astro dev
```

### Build trang web của bạn với Bun

Để build trang web của bạn với Bun làm runtime, dùng lệnh sau:

```bash
bunx --bun astro build
```

Astro sẽ xuất trang web của bạn vào thư mục `dist/`. Sau đó, bạn có thể phục vụ trang web bằng lệnh `preview`:

```bash
bunx --bun astro preview
```

## Thêm SSR với Bun

Vì Bun hỗ trợ [tương thích với API Node.js](https://bun.sh/docs/runtime/nodejs-apis), bạn có thể sử dụng bất kỳ adapter Astro nào cho [kết xuất theo yêu cầu](/vi/guides/on-demand-rendering/) với dự án Astro của mình:

```bash
bunx astro add vercel
```

## Kiểm thử trong Bun

Bun tích hợp sẵn một trình chạy kiểm thử nhanh, tương thích Jest qua lệnh `bun test`. Nếu bạn muốn sử dụng, hãy đọc [tài liệu `bun test`](https://bun.sh/docs/cli/test).

Tuy nhiên, bạn cũng có thể sử dụng Cypress hoặc Playwright để kiểm thử ứng dụng web hiện đại.

### Cypress

Cypress là công cụ kiểm thử giao diện người dùng và hướng tới mục tiêu "làm cho trải nghiệm kiểm thử trở nên thú vị và tạo ra niềm vui cho lập trình viên". Điều này cho phép bạn viết kiểm thử end-to-end cho trang Astro của mình.

Cài đặt Cypress với lệnh sau:

```bash
bun add cypress --dev
```

Để cấu hình và bắt đầu bài kiểm thử đầu tiên, hãy làm theo quy trình Cypress trong [Hướng dẫn kiểm thử Astro](/vi/guides/testing/#configuration).

### Playwright

Playwright là một framework kiểm thử end-to-end cho phép bạn kiểm thử mã Astro của mình trên tất cả các engine trình duyệt hiện đại như Chromium, WebKit, và Firefox.

Cài đặt Playwright bằng lệnh sau:

```bash
bun create playwright
```

Để tạo bài kiểm thử Playwright đầu tiên, hãy làm theo hướng dẫn trong [Hướng dẫn kiểm thử Astro](/vi/guides/testing/#create-your-first-playwright-test).
