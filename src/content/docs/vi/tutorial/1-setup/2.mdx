## Khởi động trình hướng dẫn thiết lập Astro

Cách ưu tiên để tạo một trang Astro mới là sử dụng trình hướng dẫn thiết lập `create astro`.

<Steps>
1. Trong dòng lệnh của terminal, hãy chạy lệnh sau bằng trình quản lý gói mà bạn ưa thích:

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      # tạo một dự án mới với npm
      npm create astro@latest
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      # tạo một dự án mới với pnpm
      pnpm create astro@latest
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      # tạo một dự án mới với yarn
      yarn create astro
      ```
      </Fragment>
    </PackageManagerTabs>

2. Nhập `y` để cài đặt `create-astro`.
3. Khi được hỏi bạn muốn tạo dự án ở đâu, hãy nhập tên thư mục để tạo thư mục mới cho dự án của bạn, ví dụ:
`./tutorial`

    :::note
    Một dự án Astro mới chỉ có thể được tạo trong một thư mục hoàn toàn trống, vì vậy hãy chọn tên thư mục mà chưa tồn tại!
    :::

4. Bạn sẽ thấy một danh sách ngắn các mẫu khởi đầu để lựa chọn. Sử dụng phím mũi tên (lên và xuống) để di chuyển đến mẫu minimal (trống), sau đó nhấn phím enter để xác nhận lựa chọn.

5. Khi được hỏi có muốn cài đặt các phụ thuộc không, hãy nhập `y`.

6. Khi được hỏi có muốn khởi tạo một kho git mới không, hãy nhập `y`.
</Steps>

Khi trình hướng dẫn cài đặt hoàn tất, bạn không cần sử dụng terminal này nữa. Bây giờ bạn có thể mở VS Code để tiếp tục.

## Mở dự án của bạn trong VS Code

<Steps>
7. Mở VS Code. Bạn sẽ được nhắc chọn một thư mục. Hãy chọn thư mục mà bạn đã tạo trong trình hướng dẫn thiết lập.

8. Nếu đây là lần đầu tiên bạn mở dự án Astro, bạn sẽ thấy một thông báo hỏi bạn có muốn cài đặt các tiện ích mở rộng được đề xuất không. Hãy nhấn để xem các tiện ích mở rộng được đề xuất, và chọn [tiện ích hỗ trợ ngôn ngữ Astro](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode). Tiện ích này sẽ cung cấp tô màu cú pháp và tự động hoàn thành cho mã Astro của bạn.

9. Đảm bảo rằng terminal đang hiển thị và bạn có thể nhìn thấy dấu nhắc lệnh, ví dụ:

    ```sh
    user@machine:~/tutorial$
    ```

    :::tip[Phím tắt]
    Để bật/tắt hiển thị terminal, sử dụng <kbd>Ctrl + J</kbd> (macOS: <kbd>Cmd ⌘ + J</kbd>).
    :::
</Steps>

Bạn giờ có thể sử dụng terminal tích hợp ngay trong cửa sổ này, thay vì ứng dụng Terminal trên máy tính, cho phần còn lại của hướng dẫn này.



## Chạy Astro ở chế độ phát triển

Để xem trước các tệp dự án _dưới dạng một trang web_ trong khi làm việc, bạn cần chạy Astro ở chế độ phát triển (dev).

### Khởi động máy chủ dev

<Steps>
10. Chạy lệnh để khởi động máy chủ dev của Astro bằng cách nhập vào terminal của VS Code:

    <PackageManagerTabs>
      <Fragment slot="npm">
      ```shell
      npm run dev
      ```
      </Fragment>
      <Fragment slot="pnpm">
      ```shell
      pnpm run dev
      ```
      </Fragment>
      <Fragment slot="yarn">
      ```shell
      yarn run dev
      ```
      </Fragment>
    </PackageManagerTabs>

    Bây giờ bạn sẽ thấy xác nhận trong terminal rằng Astro đang chạy ở chế độ dev. 🚀
</Steps>

## Xem bản xem trước trang web của bạn

Các tệp dự án của bạn đã chứa tất cả mã cần thiết để hiển thị một trang web Astro, nhưng trình duyệt sẽ chịu trách nhiệm hiển thị mã của bạn thành các trang web.

<Steps>
11. Nhấp vào liên kết `localhost` trong cửa sổ terminal để xem bản xem trước trực tiếp của trang web Astro mới của bạn! 

    (Astro sử dụng `http://localhost:4321` theo mặc định nếu cổng `4321` còn trống.)

    Đây là giao diện của trang web "Empty Project" mẫu của Astro trong bản xem trước trình duyệt:

    ![Một trang trắng trống với dòng chữ Astro ở phía trên.](/tutorial/minimal.png)

</Steps>

:::tip[Sử dụng máy chủ dev của Astro]

Khi máy chủ Astro đang chạy ở chế độ dev, bạn sẽ KHÔNG thể chạy các lệnh khác trong cửa sổ terminal này. Thay vào đó, khung này sẽ cung cấp phản hồi khi bạn xem trước trang web của mình.

Bạn có thể dừng máy chủ dev bất cứ lúc nào và trở về dấu nhắc lệnh bằng cách nhấn <kbd>Ctrl + C</kbd> trong terminal.

Đôi khi máy chủ dev sẽ tự động dừng khi bạn đang làm việc. Nếu bản xem trước trực tiếp bị ngừng, hãy quay lại terminal và khởi động lại máy chủ dev bằng cách nhập `npm run dev`.
:::
